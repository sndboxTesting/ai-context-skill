# Adwi — Single Source of Truth Design

> Implemented: **2026-06-17**
> Status: First version implemented (generators, validators, manifest live)

---

## The problem this solves

Before this design, four files independently maintained overlapping facts (command counts, fixture counts, port numbers, intent classes) with no validation preventing them from drifting. When values changed in code, docs had no mechanism to stay current.

Examples of drift that accumulated:
- Command count: 104 (Phase 5 note) vs 121 (directory tree) vs 172 (TOC) vs 167 (actual)
- NLU fixtures: 49 (Phase 7 note) vs 89 (directory tree) vs 96 (actual Qdrant)
- SERVICES section: showed `:1` for all ports due to a regex bug
- README header: claimed "all sections authoritative" when ~40% is manually maintained

## Truth ownership hierarchy

```
TIER 1 — CANONICAL SOURCE (never manually duplicate)
  adwi/adwi_cli.py          ← commands, intents, regex, models, NLU pipeline
  adwi/capabilities.json    ← high-level capability registry
  adwi/path_validator.py    ← security: blocked roots
  local-ai-stack/docker-compose.yml ← Docker service definitions
  ~/Library/LaunchAgents/   ← LaunchAgent schedules
  .gitignore                ← sensitive patterns

TIER 2 — GENERATED SNAPSHOT (regenerated from Tier 1 on demand)
  adwi/system_manifest.json ← generated by bin/generate-manifest
  README.md AUTO: sections  ← generated by bin/auto-update-readme

TIER 3 — HUMAN-AUTHORED DOCS (consume Tier 1/2; updated by humans)
  CLAUDE.md                 ← AI session orientation; includes current eval baseline
  docs/LLM_SYSTEM_PRIMING.md ← external-model priming; updated after major changes
  docs/OPERATOR_HANDBOOK.md ← human quick-reference

TIER 4 — ARCHIVAL / HISTORICAL (preserved as-is; not kept current)
  logs/simeval/MASTER_REPORT_v2.md ← stale eval report; see CLAUDE.md for current
  docs/NLU_REPAIR_BACKLOG.md       ← repair history; NHR items are complete
  notes/adwi-mistakes-and-fixes.md ← bug/fix log
```

## What gets generated from what

```
bin/generate-manifest
  reads:  adwi/adwi_cli.py, capabilities.json, path_validator.py,
          docker-compose.yml, LaunchAgents, .gitignore, Qdrant (live)
  writes: adwi/system_manifest.json

bin/auto-update-readme
  reads:  adwi/adwi_cli.py, capabilities.json, docker-compose.yml,
          LaunchAgents, Qdrant (live)
  writes: README.md AUTO: sections:
            MODELS, NLU, INFRA_PORTS, SERVICES, AGENTS, COMMANDS, PHASES, MONITORING
  auto-runs: before every git backup (via adwi/backup.py)

bin/validate-docs
  reads:  README.md, adwi/adwi_cli.py, capabilities.json, Qdrant (live),
          adwi/system_manifest.json
  writes: nothing (read-only checker)
  exit 0: all checks pass
  exit 1: one or more FAILs
```

## How authoritative values are computed

| Fact | Computation | Fallback |
|------|-------------|---------|
| Command count | `extract_commands()` — regex scan of adwi_cli.py + token from capabilities.json | — |
| Intent count | regex scan of `_ALL_INTENTS` list in adwi_cli.py | 109 (last known) |
| Fixture count | `GET /collections/nlu_fixtures` from Qdrant :6333 | 96 (last known) |
| Regex intent count | count `re.compile` calls in `_REGEX_INTENTS` | — |
| Model IDs | `MODEL_*` constants at top of adwi_cli.py | — |
| Services/ports | docker-compose.yml port mappings + static registry | — |
| LaunchAgent schedules | plist files in `~/Library/LaunchAgents/` | — |
| Blocked roots | `hard_blocked` list in `adwi/path_validator.py` | — |
| Python version | `python3 --version` | — |
| Eval pass rates | CLAUDE.md (manually maintained after each eval run) | — |

## Validation that prevents drift

`bin/validate-docs` checks:

1. **CMD-001**: AUTO:COMMANDS count == extract_commands() count
2. **CMD-002**: TOC command count ≈ actual (within 5)
3. **CMD-003**: §6 directory tree count ≈ actual (within 5)
4. **CMD-004**: Phase 5 count == actual (now dynamic, should always pass)
5. **NLU-001**: AUTO:NLU fixture count == Qdrant live count
6. **NLU-002**: Phase 7 fixture count == Qdrant live count (now dynamic)
7. **NLU-003**: §6 memory.py annotation == Qdrant live count
8. **NLU-004**: README intent count == _ALL_INTENTS count
9. **SVC-001**: AUTO:SERVICES has no `:1` port values
10. **MAN-001**: system_manifest.json command count == actual
11. **MAN-002**: system_manifest.json intent count == actual
12. **PRIME-001**: README header doesn't make overly broad authoritativeness claims
13. **SYNTAX-001**: adwi_cli.py compiles without errors

**When to run:** Before/after any NLU change, before commit, in CI if added.

**Expected output:** 12-13 PASS, 0 FAIL, 0-1 WARN.

## How to add a new doc fact without causing drift

**Rule:** Every fact that appears in docs must have exactly one of:
1. An AUTO: section that generates it, **or**
2. A canonical file (Tier 1) that `bin/validate-docs` checks against, **or**
3. An explicit "manually maintained" label in the doc.

**Process for new facts:**
1. Add the fact in the canonical source (adwi_cli.py, compose, etc.)
2. If it should be auto-injected: add a `SECTION_BUILDERS` entry in `bin/auto-update-readme` and an AUTO: marker in README.md
3. If it should be validated: add a check to `bin/validate-docs`
4. If it's manually maintained: label it `<!-- manually maintained -->` or add it to this file's "manually maintained" table
5. Never add the same numeric fact to multiple non-AUTO doc locations

## Trigger: when to regenerate

| Event | Action |
|-------|--------|
| Commands added/removed | Auto via `bin/auto-update-readme` before backup |
| Qdrant fixtures added | Auto via `bin/auto-update-readme` before backup |
| Docker service added | Update compose → `bin/auto-update-readme --force` + `bin/generate-manifest` |
| LaunchAgent added/changed | `bin/auto-update-readme --force` + `bin/generate-manifest` |
| Eval run completed | Manually update CLAUDE.md + `bin/generate-manifest` |
| New blocked path added | `bin/generate-manifest` |
| New doc fact added | Update validator + run `bin/validate-docs` |
| Before any git backup | `bin/auto-update-readme` runs automatically |
